üîÑ How It Works

1
Repository Analysis

Agent scans your repository structure, identifies language and build tools (pom.xml, package.json, etc.), detects test frameworks, and checks for Docker and database migrations.

2
Configuration Generation

Based on analysis, generates optimized viper_config.yaml with build settings, test configuration, security scanning (Snyk + SonarQube), deployment settings, and monitoring setup.

3
Workflow Creation

Creates .github/workflows/viper.yml that triggers VIPER pipeline on pull requests and merges. Includes proper secret management for SNYK_TOKEN and SONAR_TOKEN.

4
Validation & Recommendations

Validates the generated configuration against VIPER standards and provides recommendations for missing components like Dockerfile, tests, or migration tools.

5
Report Generation

Creates comprehensive markdown report with analysis results, generated files, next steps, and complete agent logs for transparency.
