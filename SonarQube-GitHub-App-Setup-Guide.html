Complete Configuration for Pull Request Decoration

üìã Overview

This guide walks you through creating and configuring a GitHub App to enable Pull Request Decoration in your local SonarQube instance. Pull Request Decoration allows SonarQube to:

  • Automatically comment on Pull Requests with code quality analysis
  • Show Quality Gate status directly in GitHub
  • Display code coverage and security issues on PR diffs
  • Block or warn before merging if quality standards aren't met

1 Create GitHub App

1.1 Navigate to GitHub Settings

  1. Go to your GitHub account: https://github.com/settings/apps
  2. Click on "Developer settings" (left sidebar)
  3. Click on "GitHub Apps"
  4. Click "New GitHub App" button
‚ö†Ô∏è Important: You're creating a GitHub App for your personal account, not an organization. If you want organization-wide access later, you can transfer the app.

1.2 Basic Information

Field Value Description
GitHub App name* SonarQube-Local-PR-Decoration Must be unique across GitHub
Homepage URL* http://localhost:9000 Your local SonarQube URL
Callback URL http://localhost:9000/oauth2/callback/github OAuth callback for authentication
Webhook URL http://localhost:9000/api/alm_integration/github/webhook For GitHub events (optional for basic setup)
Webhook Secret Leave blank for now Only needed for GitHub Code Scanning
üí° Note: Since this is a local instance, GitHub won't be able to reach your webhook URL. This is fine for manual analysis. If you later deploy SonarQube to a public server, update these URLs.

2 Collect App Credentials

You should now be on your GitHub App's settings page. Let's collect the required information.

2.1 Get App ID

Location: Top of the page, under "About"
Look for: "App ID: 123456"
Copy this number - you'll need it for SonarQube configuration.

2.2 Get Client ID

Location: In the "General" section
Look for: "Client ID"
Format: Iv1.a1b2c3d4e5f6g7h8
Copy this value

2.3 Generate Client Secret

  1. Find the "Client secrets" section
  2. Click "Generate a new client secret"
  3. A secret will be generated (format: abc123...xyz789)
  4. ‚ö†Ô∏è Copy immediately! - You won't be able to see it again
‚ö†Ô∏è Critical: Save this Client Secret immediately. GitHub will only show it once. If you lose it, you'll need to generate a new one.

2.4 Generate Private Key

  1. Scroll down to "Private keys" section
  2. Click "Generate a private key"
  3. A .pem file will download automatically
  4. Keep this file safe - you'll paste its contents into SonarQube
# To view the private key contents on macOS: cat ~/Downloads/your-app-name.*.private-key.pem # The file looks like this: -----BEGIN RSA PRIVATE KEY----- MIIEpAIBAAKCAQEA... ...multiple lines of base64 encoded key... ... -----END RSA PRIVATE KEY-----
‚úÖ Credentials Collected! You should now have:
  • ‚úì App ID (6-7 digit number)
  • ‚úì Client ID (starts with "Iv1.")
  • ‚úì Client Secret (long string)
  • ‚úì Private Key (.pem file contents)

3 Install App on Repository

3.1 Install the App

  1. On your GitHub App settings page, click "Install App" (left sidebar)
  2. Click the "Install" button next to your account
  3. Choose installation option:
    • All repositories - Gives access to all your repos (easier)
    • Only select repositories - Choose specific repos (more secure)
  4. If choosing specific repos, select: testenvfor_sonar
  5. Click "Install"
üí° Recommendation: For learning purposes, install on "All repositories". You can always revoke access later from Settings ‚Üí Applications ‚Üí Installed GitHub Apps.
‚úÖ Installation Complete! Your GitHub App now has access to your repository.

üîß Troubleshooting

Common Issues

Issue: "Invalid Private Key"
Solution: Ensure you copied the entire .pem file contents including BEGIN/END lines. No extra spaces.
Issue: "Failed to connect to GitHub API"
Solution: Check your internet connection. Verify GitHub API URL is exactly: https://api.github.com/
Issue: "App not installed"
Solution: Make sure you completed Step 3 - installing the app on your repository.
Issue: PR decoration not appearing
Solution:
  • Verify the app has "Checks" and "Pull requests" write permissions
  • Ensure analysis ran with correct PR parameters
  • Check SonarQube logs: ~/Downloads/sonarqube-26.2.0.119303/logs/web.log

Verification Checklist

  • ‚úì GitHub App created with correct permissions
  • ‚úì Private key generated and saved
  • ‚úì App installed on repository
  • ‚úì SonarQube configuration saved successfully
  • ‚úì Project created in SonarQube
  • ‚úì Analysis token generated
  • ‚úì Local analysis runs successfully
